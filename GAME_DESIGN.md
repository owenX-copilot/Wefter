# Wefter — 游戏设计大纲

> 基于 SerendipUDP 的俯视角实时迷宫游戏。  
> 世界在变，你在其中切出一片属于自己的陆地。

---

## 一句话描述

玩家在由真实节点网络持续扰动的无限迷宫中探索，将短暂冻结的地图碎片锚定为永久领土，向混沌要一片立足之地。

---

## 核心张力

| 混沌 | 秩序 |
|------|------|
| 未锚定区块随 seed 变化，迷宫持续重构 | 锚定后永久固定，是安全区和补给地 |
| 敌人属性随 seed 随机生成 | 玩家通过积累领土获得成长 |
| 相遇时机不可控 | 锚定哪块地图是玩家的主动选择 |

---

## 世界结构

### 区块系统

- 世界由 **N×N 的区块网格**构成，每个区块是一张独立的迷宫（例如 20×20 格）
- 区块坐标 (0,0) 是玩家**初始锚定区块**，结构固定，作为家园/基地
- 每个区块对应 SerendipUDP 的一个 seed 值，seed 变化时迷宫重新生成
- **出口位置固定**（每条边中点），保证区块间始终联通；seed 只控制内部路径

### 区块状态

```
未解放（变动中）→  解放（冻结快照）→  锚定（永久固定）
                               ↑
                        仅持有"钥匙"，
                        不锚定则继续变化
```

- **未解放**：迷宫随 seed 实时重构，敌人随 seed 属性变化，资源随机刷新
- **解放**：完成解放条件的瞬间，生成该时刻的地图快照，玩家拿到一把"地图钥匙"
- **锚定**：在家园界面消耗钥匙，将快照永久写入该坐标——这块地图从此不再变化

### 解放条件（三选一，由 seed 决定区块类型）

| 类型 | 条件 | 解放后额外奖励 |
|------|------|----------------|
| 荒野区块 | 收集区块内所有资源 | 资源数量根据 seed 随机 |
| 商队区块 | 进入商店区域完成一笔交易 | 可购买装备/消耗品 |
| 敌营区块 | 击败区块内所有敌人 | 掉落材料，seed 越偏越强 |

---

## 锚定系统（核心决策）

解放区块后，玩家持有该区块的地图快照（"钥匙"）。

**玩家需要选择：**
- **锚定它**：消耗钥匙，将这块地图永久固定在世界上
- **放弃它**：让它继续随机变化，等待更好的版本出现

**为什么不是每块都锚定：**
- 钥匙需要在家园兑换，带回途中可能遭遇
- 锚定位置有讲究：好的商队区块应该放在靠近家园的位置
- 某些地图出口位置太差，宁可等一个新的

## 成长系统

### 玩家成长

- 击杀敌人 → 经验 → 被动技能解锁（移速、视野、攻击等）
- 收集材料 → 在商队区块或家园升级装备
- 无等级数字，仅通过装备词条体现强度

### 世界成长

- 锚定的区块越多，"领土"越大
- 领土内资源自动再生（每隔若干秒）
- 家园可解锁建筑（仓库、传送点），花费锚定区块获得的材料

### 难度曲线

- 距离 (0,0) 越远的区块，seed 派生出的敌人属性上限越高
- 没有人为设计的"关卡"，距离即难度

## seed 来源设计

游戏内部通过统一的 seed 提供层获取随机源，与上层逻辑解耦。

### 接口规范（`/api/seed`）

响应格式（JSON）：
```json
{
  "seeds": [
    {"id": "node-0", "value": 12345678, "updated_at": 1709712000},
    {"id": "node-1", "value": 87654321, "updated_at": 1709712005}
  ]
}
```

- `id`：节点标识符，对应世界上不同象限的区块
- `value`：32 位无符号整数，作为该象限的 seed 输入 LCG
- `updated_at`：Unix 时间戳，游戏通过对比上次值检测 seed 是否变化

游戏定时轮询（每 10 秒）此接口，检测到 `value` 变化时触发对应区块重构。

### MVP 模式：本地 LCG 模拟

MVP 阶段游戏内置一个假的 seed 提供层，完全不依赖 SerendipUDP：

```javascript
// mock_seed_provider.js
// 每隔随机 30~120 秒，某个 seed 自动漂移一次，模拟节点相遇
function getSeeds() {
  return {
    seeds: [
      { id: "node-0", value: localSeed0, updated_at: Date.now() / 1000 },
      { id: "node-1", value: localSeed1, updated_at: Date.now() / 1000 }
    ]
  };
}
```

接口形状与真实 `/api/seed` 完全一致，上层游戏逻辑无需修改。

### 升级路径

当 SerendipUDP 节点就绪后，只需在游戏配置里将 seed 来源从内置模拟切换到真实接口：

```javascript
// config.js
const SEED_SOURCE = "local";       // MVP：使用本地 LCG 模拟
// const SEED_SOURCE = "http://localhost:7777/api/seed";  // 接入真实节点
```

**真实节点的额外意义：**
- seed 变化发生在真实 UDP 包到达的时刻，不可预测，无法被游戏策划控制
- 只需 2-3 个活跃节点，每个节点对应世界上不同象限的区块
- 节点发生 SYNC 事件 = 游戏里某个象限的迷宫在某个真实时刻突变

## 技术方案

| 模块 | 方案 |
|------|------|
| 游戏引擎 | Phaser.js 3（CDN 引入，无需构建） |
| 迷宫生成 | LCG → 递归分割法或 BSP，seed 作为随机源 |
| 地图存储 | 锚定的区块保存为 localStorage JSON |
| seed 来源（MVP） | 游戏内置本地 LCG 模拟，接口形状与真实 `/api/seed` 相同 |
| seed 来源（完整版） | SerendipUDP `http_server.py` 新增 `/api/seed` 接口 |
| 部署（MVP） | 单个 HTML 文件，任意静态服务或直接双击打开 |
| 部署（完整版） | 挂在 SerendipUDP 现有的静态文件服务下 |

## 最小可玩原型目标

> 玩家能在随机生成的迷宫里移动，拿完资源后解放区块，拿到钥匙，回到 (0,0) 锚定它，然后去下一个区块。地图用 hardcode seed，没有敌人，没有战斗。

这一步做完，游戏的核心感受就能验证了。
