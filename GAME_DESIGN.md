# Wefter — 游戏设计大纲

> 基于 SerendipUDP 的俯视角实时迷宫游戏。  
> 世界在变，你在其中切出一片属于自己的陆地。

---

## 一句话描述

玩家在由真实节点网络持续扰动的无限迷宫中探索，将短暂冻结的地图碎片锚定为永久领土，向混沌要一片立足之地。

---

## 核心张力

| 混沌 | 秩序 |
|------|------|
| 未锚定区块随 seed 变化，迷宫持续重构 | 锚定后永久固定，是安全区和补给地 |
| 敌人属性随 seed 随机生成 | 玩家通过积累领土获得成长 |
| 相遇时机不可控 | 锚定哪块地图是玩家的主动选择 |

---

## 世界结构

### 区块系统

- 世界由 **N×N 的区块网格**构成，每个区块是一张独立的迷宫（例如 20×20 格）
- 区块坐标 (0,0) 是玩家**初始锚定区块**，结构固定，作为家园/基地
- 每个区块对应 SerendipUDP 的一个 seed 值，seed 变化时迷宫重新生成
- **出口位置固定**（每条边中点），保证区块间始终联通；seed 只控制内部路径

### 区块状态

```
未解放（变动中）→  解放（冻结快照）→  锚定（永久固定）
                               ↑
                        仅持有"钥匙"，
                        不锚定则继续变化
```

- **未解放**：迷宫随 seed 实时重构，敌人随 seed 属性变化，资源随机刷新
- **解放**：完成解放条件的瞬间，生成该时刻的地图快照，玩家拿到一把"地图钥匙"
- **锚定**：在家园界面消耗钥匙，将快照永久写入该坐标——这块地图从此不再变化

### 解放条件（三选一，由 seed 决定区块类型）

| 类型 | 条件 | 解放后额外奖励 |
|------|------|----------------|
| 荒野区块 | 收集区块内所有资源 | 资源数量根据 seed 随机 |
| 商队区块 | 进入商店区域完成一笔交易 | 可购买装备/消耗品 |
| 敌营区块 | 击败区块内所有敌人 | 掉落材料，seed 越偏越强 |

---

## 锚定系统（核心决策）

解放区块后，玩家持有该区块的地图快照（"钥匙"）。

**玩家需要选择：**
- **锚定它**：消耗钥匙，将这块地图永久固定在世界上
- **放弃它**：让它继续随机变化，等待更好的版本出现

**为什么不是每块都锚定：**
- 钥匙需要在家园兑换，带回途中可能遭遇
- 锚定位置有讲究：好的商队区块应该放在靠近家园的位置
- 某些地图出口位置太差，宁可等一个新的

---

## 成长系统

### 玩家成长

- 击杀敌人 → 经验 → 被动技能解锁（移速、视野、攻击等）
- 收集材料 → 在商队区块或家园升级装备
- 无等级数字，仅通过装备词条体现强度

### 世界成长

- 锚定的区块越多，"领土"越大
- 领土内资源自动再生（每隔若干秒）
- 家园可解锁建筑（仓库、传送点），花费锚定区块获得的材料

### 难度曲线

- 距离 (0,0) 越远的区块，seed 派生出的敌人属性上限越高
- 没有人为设计的"关卡"，距离即难度

## seed 来源设计

游戏内部通过统一的 seed 提供层获取随机源，与上层逻辑解耦。

### 接口规范（`/api/seed`）

响应格式（JSON）：
```json
{
  "seeds": [
    {"id": "node-0", "value": 12345678, "updated_at": 1709712000},
    {"id": "node-1", "value": 87654321, "updated_at": 1709712005}
  ]
}
```

- `id`：节点标识符，对应世界上不同象限的区块
- `value`：32 位无符号整数，作为该象限的 seed 输入 LCG
- `updated_at`：Unix 时间戳，游戏通过对比上次值检测 seed 是否变化

游戏定时轮询（每 10 秒）此接口，检测到 `value` 变化时触发对应区块重构。

### MVP 模式：本地 LCG 模拟

MVP 阶段游戏内置一个假的 seed 提供层，完全不依赖 SerendipUDP：

```javascript
// mock_seed_provider.js
// 每隔随机 30~120 秒，某个 seed 自动漂移一次，模拟节点相遇
function getSeeds() {
  return {
    seeds: [
      { id: "node-0", value: localSeed0, updated_at: Date.now() / 1000 },
      { id: "node-1", value: localSeed1, updated_at: Date.now() / 1000 }
    ]
  };
}
```

接口形状与真实 `/api/seed` 完全一致，上层游戏逻辑无需修改。

### 升级路径

当 SerendipUDP 节点就绪后，只需在游戏配置里将 seed 来源从内置模拟切换到真实接口：

```javascript
// config.js
const SEED_SOURCE = "local";       // MVP：使用本地 LCG 模拟
// const SEED_SOURCE = "http://localhost:7777/api/seed";  // 接入真实节点
```

**真实节点的额外意义：**
- seed 变化发生在真实 UDP 包到达的时刻，不可预测，无法被游戏策划控制
- 只需 2-3 个活跃节点，每个节点对应世界上不同象限的区块
- 节点发生 SYNC 事件 = 游戏里某个象限的迷宫在某个真实时刻突变

## 技术方案

| 模块 | 方案 |
|------|------|
| 游戏引擎 | Phaser.js 3（CDN 引入，无需构建） |
| 迷宫生成 | LCG → 递归分割法或 BSP，seed 作为随机源 |
| 地图存储 | 锚定的区块保存为 localStorage JSON |
| seed 来源（MVP） | 游戏内置本地 LCG 模拟，接口形状与真实 `/api/seed` 相同 |
| seed 来源（完整版） | SerendipUDP `http_server.py` 新增 `/api/seed` 接口 |
| 部署（MVP） | 单个 HTML 文件，任意静态服务或直接双击打开 |
| 部署（完整版） | 挂在 SerendipUDP 现有的静态文件服务下 |

## 商店系统设计

### 一次性商店（Shop 区块）

每个 Shop 区块是地图上随机分配的商队驻扎点，提供**一次性购物机会**，购买后区块被解放。

#### 区块布局

- 迷宫结构与 Wild/Enemy 相同（随 seed 生成）
- 中心格（MID, MID）放置**商人 NPC** 标识（橙色 `$` 图标）
- 走到中心格，按 **F** 键进入商店界面

#### 购物流程

1. 进入商店界面，展示 **3 件随机商品**（3选1，只能购买1件）
2. 玩家选择并支付 🪙 金币
3. 购买完成 → 区块**立即解放**，获得一把标注"商店"的地图钥匙
4. 钥匙为 `MapKey`，`label` 格式：`🏪 商店 (cx, cy)`

> 若金币不足以购买任何商品，可选择**放弃**（不购买也不解放，区块维持原状）。  
> 玩家重新进入该区块时，商品随机内容**不变**（由 seed 固定）。

---

### 固定商店（锚定后的 Shop 区块）

当玩家携带商店钥匙回家园，选择将其**锚定**到世界坐标，该区块变为**永久商店据点**：

#### 视觉形态

- 布局与家园类似：周围迷宫墙全部消失，全部变为 **Path 地板**（灰蓝色，区别于家园绿色）
- 中心保留商人 NPC，四周可自由通行

#### 刷新机制

- 固定商店每 **10 分钟**自动刷新一批新商品（3选1）
- 玩家在该区块中心按 **F** 随时查看/购买当前商品
- 每次刷新消耗一个**内部计时器**，与存档时间对比推算（离线期间不累计）
- 每次刷新购买次数重置为 1（每轮只能买1件）
- HUD 显示距下次刷新的倒计时（仅站在该区块时显示）

---

### 商品池设计

商品分为三类，每次从全池中随机抽取3件不重复的展示：

#### 消耗品

| 商品 | 价格 | 效果 |
|------|------|------|
| 急救包 | 8🪙 | 立即回复 5 HP |
| 储备粮 | 15🪙 | healBank +50 |
| 烟雾弹 | 6🪙 | 当前区块所有敌人 3 回合无法感应玩家 |

#### 增益（持续到死亡/切区块）

| 商品 | 价格 | 效果 |
|------|------|------|
| 加速符 | 10🪙 | 当前区块移动速度 +50%（1次区块） |
| 护盾 | 12🪙 | 下次受伤免疫（1次） |

#### 永久升级（本次游戏存档有效）

| 商品 | 价格 | 效果 |
|------|------|------|
| 火力强化I | 20🪙 | 攻击力 +1（子弹伤害 2→3） |
| 最大血量+5 | 25🪙 | PLAYER_MAX_HP 永久 +5 |
| 侦测压制 | 18🪙 | 所有 Scout 感应半径 -2 |

> 初期实现建议先只做**消耗品**，永久升级后续迭代加入。

---

### 技术要点

- **商品随机**：由该区块 `seed` 派生，保证同一区块每次进入看到相同商品（一次性购买后不再显示）
- **固定商店刷新**：`shopRefreshAt` 时间戳存入 `ChunkData`（或 `liberatedData`），与当前时间对比
- **UI 层**：弹出覆盖层（类似 AnchorScene），3 个商品格子，点击选择后确认购买
- **金币检查**：不满足价格的商品显示为灰色不可点击

## 敌人系统设计（阶段三细化）


### 战斗机制总述

采用 **Roguelike 撞击+弹幕混合** 模式，战斗发生在迷宫格子内，无独立战斗场景：

- **玩家攻击**：与敌人处于同一条可达路径的若干步内时，自动朝最近的敌人开火（无需手动按键），子弹按格子直线飞行，不可穿墙拐弯
- **近战撞击**：玩家走入敌人格子 = 近战攻击（可补刀或应急）
- **回合感**：玩家每走一步，所有敌人也同步移动/行动一步（回合制节奏，无缝融入移动）

### 三种敌人

---

#### 🔵 巡逻者（Scout）

**定位**：信息型单位，本身无害，但它是整张地图的眼睛

| 属性 | 值 |
|------|----|
| 血量 | 低（1~2 击毙） |
| 伤害 | **无** |
| 移动 | 每回合从当前可通行方向中随机选一个移动（不寻路，维护可移动方向队列） |
| 感应方式 | **半径感应（穿墙欧氏距离）**：与玩家距离 ≤ R（约 5 格）时触发 |

**发现玩家后：**
- **不追击**，继续随机游荡（行为不变）
- 广播玩家的精确坐标给同区块所有狙击者
- 广播**持续到巡逻者死亡或玩家走出感应半径**为止（自然解除）

**设计意图**：玩家主动跑出感应半径即可解除广播，所以优先打巡逻者是选择而非必须

---

#### 🔴 追击者（Chaser）

**定位**：普通近战威胁，压迫玩家空间

| 属性 | 值 |
|------|----|
| 血量 | 中 |
| 伤害 | 正常（1点） |
| 初始状态 | 静止不动 |
| 感应方式 | **路径感应（BFS 最短步数）**：玩家与追击者迷宫最短路径 ≤ N 步（约 10 步）时激活 |

**激活后：**
- 每回合沿 BFS 最短路径走 1 步
- 每走 3 步，额外多走 1 步（模拟"冲刺加速"）
- 始终沿迷宫可达路径追踪，不穿墙

> **备选简化**：若实现复杂度不值得，可改为"固定靶"——静止不动，玩家走入相邻格触发撞击伤害，当作普通障碍型小怪

**设计意图**：走廊战术，利用迷宫结构拉开距离或卡位单通道

---

#### 🟡 狙击者（Sniper）

**定位**：高威胁远程单位，不移动，依赖视野优势

| 属性 | 值 |
|------|----|
| 血量 | 低 |
| 伤害 | 低（0.5点，两发才打掉1血） |
| 攻速 | 低（每 3 回合射击一次） |
| 移动 | **不移动** |
| 感应 | **路径感应（BFS）**：玩家最短路径 ≤ M 步（约 15 步）时激活 |

**实现优化（预计算攻击范围）：**  
区块生成时，对每个狙击者位置做一次射线扫描，将其横/竖直线上无墙可达的所有格子存入 `Set<"x,y">`（热力图），运行时判断玩家是否在攻击范围只需 O(1) 查表。

**正常状态（无广播）：**
- 只攻击在其**预计算可见格集合**内的玩家（同直线无墙）
- 子弹沿直线飞行，遇墙停止

**巡逻者广播激活期间：**
- 无视距离、无视障碍物、直接朝玩家当前坐标穿墙射击（"透视炮"）
- 广播持续到**巡逻者死亡**或**玩家走出该巡逻者的感应半径**为止

**设计意图**：玩家打不过可以跑出巡逻者半径自然解除穿墙效果；保留狙击者是持续的定点威胁

---

### 区块内数量（由 seed 派生）

| 距离 (0,0) | 巡逻者 | 追击者 | 狙击者 |
|-----------|--------|--------|--------|
| 1~3 格     | 1      | 1~2    | 0~1    |
| 4~7 格     | 1~2    | 2~3    | 1      |
| 8+ 格      | 2~3    | 3~4    | 1~2    |

### 解放条件

击败区块内**全部敌人**（巡逻者 + 追击者 + 狙击者）→ 区块解放，可获取地图快照钥匙
